# AIProvider 前端 UI 整改清单

本清单用于约束全局 UI 重构。视觉调整不得破坏现有业务状态、异步流程、队列持久化或页面可达性。

## 本轮实施结果（2026-07-19）

- 已完成：桌面端分组文字导航、明确选中态、统一页头与页面说明；`1280×720` 下导航无需滚动且所有一级入口可见。
- 已完成：非工坊页面移除全局雾层与竞争性装饰，统一语义表面、表单边界、焦点、弱文字可读性和控件最低字号。
- 已完成：Prompt 方案列表从 clickable `div` 重构为无嵌套冲突的原生按钮结构。
- 已完成：监控中心健康状态条溢出、内容运营错误提示遮挡页头和双层滚动、首页重复版本号等实测问题。
- 已完成：空/非 JSON 服务响应统一转为带 HTTP 状态的中文错误，不静默吞掉失败。
- 已完成：UI 门禁和相关页面测试通过，生产构建通过；全量 151 项中 150 项通过，唯一失败是现有本机图库缺少非 ASCII 文件名测试夹具，与本轮 UI 及图像工坊代码无关。
- 边界确认：`ComfyLocalWorkbench*`、`WorkflowPanel*` 及其内部逻辑/样式零改动；工坊保持 76px 原导航轨，回归结果为 `1280/1280` 无横向溢出。

## 0. 本轮明确边界

- [ ] 不修改 `ComfyLocalWorkbench.jsx`、`ComfyLocalWorkbench.css`、`WorkflowPanel.jsx`、`WorkflowPanel.css` 及其相关测试。
- [ ] 不改变图像工坊内部布局、控件、样式、状态机、队列、轮询、画廊和交互逻辑。
- [ ] 全局应用外壳、一级导航和公共页头可以重构；图像工坊只被新的公共外壳包裹，内部保持原样。

## 1. 全局验收

- [ ] 仅面向桌面端，所有一级工作区在 `1280×720` 可达且主体无横向溢出。
- [ ] 左侧一级导航同时显示图标与文字；当前页不能只靠颜色区分。
- [ ] 首页仍可进入图像工坊，并固定显示前端、后端两行版本号。
- [ ] 页面背景、卡片、正文、弱文字、边框与主按钮统一消费 `uiTheme` 语义变量。
- [ ] 正文目标对比度不低于 4.5:1；重要边界、状态与焦点目标不低于 3:1。
- [ ] 所有可交互控件使用原生交互元素，具备可见焦点和准确名称。
- [ ] 搜索框全部复用 `UiSearchField`，右侧搜索图标，不增加无意义提交按钮。
- [ ] 业务页面移除遮盖全局内容的低透明雾层；品牌装饰不得与状态、标题、主操作竞争。
- [ ] 主操作、次操作、危险操作、禁用操作具有稳定且一致的层级。
- [ ] 加载、空数据、异常、离线、无权限、成功与进行中状态能被明确区分。

### 主题对比度基线（当前实测）

- [ ] 默认主题 `--text-muted` / `--bg-card` 当前为 4.16:1，需要提升到至少 4.5:1。
- [ ] 15 套预设中仅 `github-dark` 的弱文字/卡片达到 4.5:1，需逐套修正弱文字颜色。
- [ ] 15 套预设的 `--border-normal` / `--bg-card` 均低于 3:1；表单、焦点、选中和重要分隔不能继续依赖该边框独自表达状态。
- [ ] 普通装饰分隔线可保持低权重，但交互控件边界和状态边界必须使用可达到 3:1 的独立语义变量或额外形状提示。

- [ ] URL 与 `view` 的双向映射、浏览器前进/后退恢复保持有效。
- [ ] Prompt 方案页与词条管理页共享一级导航选中状态。

### 图像工坊（只做回归验证，不实施改动）

- [ ] 保留启动器 `checking / ready / missing` 状态。
- [ ] 保留 ComfyUI `running / stopped / path-unconfigured` 状态及启动、停止、重启前置条件。
- [ ] 保留平台未配置、Bridge 启动、超时、错误与操作成功反馈。
- [ ] 生成按钮继续受 `running + token + activeWorkflow + batchScheduling` 约束。
- [ ] 工作流未加载、输入图片缺失、参考图缺失、区域编辑数据无效等校验继续生效。
- [ ] 保留 Bridge 队列、ComfyUI 队列、运行、取消中、失败、完成及外部任务状态。
- [ ] 保留任务进度节点、总进度、运行时间、取消、失败任务移除与任务详情。
- [ ] 保留本机图片、待处理、我的资产、回收站的独立数据源、筛选、分页和选择模式。

## 3. 页面整改重点

- [ ] 首页：建立清晰入口与产品身份，同时保留沉浸式主视觉和版本信息。
- [ ] 图像工坊：不做视觉或结构整改，只验证新的公共外壳未破坏现有页面。
- [ ] Prompt 管理：降低纵向滚动和重复搜索密度，强化保存与危险操作区分。
- [ ] 视频编辑：空工程突出导入，进入编辑态后恢复高密度专业工具。
- [ ] 监控中心：先展示整体健康与异常，再展示资源、网络和费用明细。
- [ ] 远程 Codex：根据未授权、未登录、已登录无对话、对话运行中切换布局重点。
- [ ] 系统设置：长路径支持可读展示，统一保存、刷新、打开、新建等动作层级。
- [ ] 其他一级工作区：统一导航、页头、主题、焦点、空状态和错误状态。

## 4. 参考原则

- Material navigation rail：主要目的地应同时提供图标与文字标签。
- Carbon empty states：空状态说明原因，并给出一个明确的下一步动作。
- Atlassian navigation：稳定侧栏、模式复用与渐进披露，降低跨工作区认知成本。
- WCAG 2.2：正文 4.5:1、非文本边界与焦点 3:1，并提供清晰可见的焦点指示。
