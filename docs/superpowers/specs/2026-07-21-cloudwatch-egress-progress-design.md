# CloudWatch 出站流量进度条设计

## 目标

让监控中心的 CloudWatch 网络卡片直观展示 AWS 每月 100GB 免费互联网出站额度，并消除“下载/上传”在用户视角与服务器视角之间的歧义。

## 展示

- 实时指标改名为“入站速率（用户上传）”和“出站速率（用户下载）”。
- 月度指标改名为“本月入站”和“本月出站”。
- 在卡片内新增“免费出站额度”主进度条，数值使用后端同一 AWS 流量快照中的 `traffic.usedBytes / traffic.totalBytes`。
- 进度条同时显示已用量、100GB 总额度、百分比和剩余量。
- 70% 以上使用警告色，90% 以上使用危险色，与内存和磁盘卡片一致。
- 入站不计入 100GB 百分比；数据不可用时显示“—”，不构造比例。

## 卡片高度

桌面布局不再用剩余视口高度强制拉伸底部四张资源卡。容量网格改为内容自然高度，服务器内存、系统磁盘、CloudWatch 网络和 AWS 费用卡顶部对齐；加载态和数据不可用状态同样保持紧凑。移动端继续单列自然高度，不设置固定高度。

## 数据边界

当前数据源是指定 EC2 实例的 CloudWatch `NetworkOut`，代表实例出站参考值。页面文案不再声称它是全账户精确可计费互联网出站量；标注为“CloudWatch 实例出站参考”。实时速率仍是最近 30 分钟平均值，并明确显示为“近 30 分钟平均”。

## 改动范围与验证

只修改 `MonitorCenter.jsx`、`MonitorCenterEnhancements.css` 和 `MonitorCenter.test.jsx`。测试验证用户视角文案、出站百分比、剩余量、入站未被计入额度，以及底部卡片不再强制拉伸；同时通过 UI 门禁、前端定向测试、构建和桌面/移动端渲染检查。
